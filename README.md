## Requirements

Python 3.10 or higher is required. Install all dependencies with:

```bash
pip install -r requirements.txt
```

Key packages: `pandas`, `numpy`, `scikit-learn`, `xgboost`, `shap`, `matplotlib`.

## Data

Three raw datasets are required and should be placed in `data/raw/`:

| File | Source | Description |
|------|--------|-------------|
| `weather_data_original.txt` | KNMI | Daily weather observations from Gilze-Rijen Airport (2010–2025) |
| `wildfire_data.csv` | VRMWB | Recorded wildfire incidents in Central and West Brabant (2011–2025) |
| `calendar_features.csv` | Compiled manually | Daily holiday and vacation indicators for NL, BE and DE (2011–2025) |

The weather and calendar datasets are publicly available. The wildfire dataset was obtained with permission from VRMWB and is not publicly distributed.

## How to Run

Run the complete pipeline with:

```bash
python src/main.py
```

This will:
1. Clean and engineer features from all three raw datasets
2. Merge them into training (2011–2023) and test (2024–2025) sets
3. Run walk-forward cross-validation across four models and two target types
4. Evaluate performance on the out-of-sample test set
5. Compute SHAP feature importance for all non-dummy models
6. Save all results to `data/output/`

Individual modules can also be run directly for debugging:

```bash
python src/data_prep_v2.py   # Data preparation only
python src/trainers.py       # Training on example dataset only
python src/shap_analyser.py  # SHAP analysis on example dataset only
```

## Configuration

Key settings in `main.py` that can be adjusted:

| Setting | Location | Default | Description |
|---------|----------|---------|-------------|
| `window_length` | `WeatherPrep.feature_engineering()` | `[8, 16, ..., 80]` | Sliding window sizes in days |
| `models` | Training loop | All four | Which trainer classes to run |
| `target` | Training loop | Both | `binary_wf` and/or `numeric_wf` |
| `current_dataset` | `main()` | `train_data` | Switch to `example_data` for faster runs |

## Reproducing Results

To reproduce the exact results reported in the thesis:

1. Place the three raw datasets in `data/raw/`
2. Create the required output directories: `data/processed/`, `data/output/`, `data/output/test_predictions/`, `figures/`
3. Run `python src/main.py`

All random states are set to `2026` for reproducibility.
